### [ Сценарий 1 - Создание учреждения и переход к данным ]

1. Пользователь переходит на страницу "Учреждения"
2. Нажимает на кнопку "Создать учреждение"
3. Вводит название учреждения в поле "Название"
4. Нажимает на кнопку "Создать"
5. Ожидает сохранения и автоматического перехода к списку учреждений
6. Находит созданное учреждение в списке
7. Нажимает на кнопку "Открыть" у нужного учреждения и переходит на страницу с вкладками


### [ Сценарий 2 - Заполнение информации о преподавателях ]

1. Пользователь открывает учреждение и переходит на вкладку "Преподаватели"
2. Нажимает на кнопку "Новый преподаватель"
3. В открывшемся диалоге вводит полное имя в поле "Полное имя"
4. При необходимости вводит предмет в поле "Предмет"
5. Нажимает на кнопку "Создать"
6. При необходимости нажимает на иконку "Назначить уроки" у карточки преподавателя и отмечает чекбоксами нужные уроки, затем нажимает "Назначить"
7. Повторяет шаги 2–6 для всех преподавателей


### [ Сценарий 3 - Заполнение информации об аудиториях ]

1. Пользователь открывает учреждение и переходит на вкладку "Аудитории"
2. Нажимает на кнопку "Новая аудитория"
3. В диалоге вводит название в поле "Название"
4. Вводит вместимость в поле "Вместимость"
5. При необходимости заполняет поля "Тип аудитории" и "Оборудование"
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 2–6 для всех аудиторий


### [ Сценарий 4 - Заполнение информации о классах ]

1. Пользователь открывает учреждение и переходит на вкладку "Классы"
2. Нажимает на кнопку "Новая группа класса"
3. В диалоге вводит название класса в поле "Название"
4. Вводит количество студентов в поле "Количество студентов"
5. Нажимает на кнопку "Создать"
6. Убеждается, что группа отобразилась в списке
7. Повторяет шаги 2–6 для всех классов


### [ Сценарий 5 - Заполнение информации о студентах ]

1. Пользователь открывает учреждение и переходит на вкладку "Студенты"
2. Нажимает на кнопку "Новый студент"
3. В диалоге выбирает группу класса в выпадающем списке "Группа класса"
4. Вводит ФИО в поле "Полное имя"
5. При необходимости вводит номер в поле "Номер студента"
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 2–6 для всех студентов


### [ Сценарий 6 - Заполнение информации об уроках ]

1. Пользователь открывает учреждение и переходит на вкладку "Уроки"
2. Нажимает на кнопку "Новый урок"
3. В диалоге вводит название в поле "Название"
4. При необходимости вводит код предмета в поле "Код предмета"
5. Задаёт длительность в минутах в поле "Длительность (минуты)"
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 2–6 для всех уроков


### [ Сценарий 7 - Настройка потоков и учебных групп ]

1. Пользователь открывает учреждение и переходит на вкладку "Потоки"
2. Нажимает на кнопку "Новый поток", вводит название и отмечает чекбоксами группы классов, нажимает "Создать"
3. Переходит на вкладку "Учебные группы"
4. Нажимает на кнопку "Новая учебная группа"
5. В диалоге выбирает поток в выпадающем списке "Поток", вводит название, отмечает чекбоксами студентов
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 4–6 для всех учебных групп


### [ Сценарий 8 - Привязка уроков к группам ]

1. Пользователь открывает учреждение и переходит на вкладку "Привязка уроков"
2. В блоке "Группы классов" находит нужную группу
3. Нажимает на иконку "Назначить уроки" у карточки группы
4. В диалоге отмечает чекбоксами уроки и при необходимости задаёт количество для каждого урока
5. Нажимает на кнопку "Назначить"
6. Аналогично назначает уроки для учебных групп в блоке "Учебные группы"
7. Повторяет шаги 2–6 для всех групп, которым нужно задать набор уроков


### [ Сценарий 9 - Настройка временных слотов ]

1. Пользователь открывает учреждение и переходит на вкладку "Временные слоты"
2. Нажимает на кнопку "Новый временной слот"
3. В диалоге выбирает день недели в выпадающем списке "День недели"
4. Задаёт "Время начала" и "Время окончания"
5. Вводит номер слота в поле "Номер слота"
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 2–6 для всех слотов учебной недели


### [ Сценарий 10 - Настройка ограничений (при необходимости) ]

1. Пользователь открывает учреждение и переходит на вкладку "Ограничения"
2. Нажимает на кнопку "Новое ограничение"
3. Вводит тип ограничения в поле "Тип ограничения"
4. Задаёт приоритет (0–10) в поле "Приоритет"
5. Вводит JSON в поле "Данные ограничения (JSON)"
6. Нажимает на кнопку "Создать"
7. Повторяет шаги 2–6 для всех нужных ограничений


### [ Сценарий 11 - Создание и генерация расписания ]

1. Пользователь открывает учреждение и переходит на вкладку "Расписания"
2. Нажимает на кнопку "Новое расписание"
3. На странице создания выбирает учреждение (если не подставлено), вводит название расписания и при необходимости учебный период
4. Нажимает на кнопку "Создать расписание"
5. Переходит на страницу созданного расписания со статусом "Черновик"
6. Нажимает на кнопку "Создать расписание" (генерация) и подтверждает действие
7. Ожидает завершения генерации и появления сообщения о количестве созданных записей


### [ Сценарий 12 - Экспорт расписания ]

1. Пользователь генерирует расписание и дожидается смены статуса на "Сгенерировано"
2. На странице расписания находит кнопку "Экспорт PDF"
3. Нажимает на кнопку "Экспорт PDF"
4. Дожидается формирования файла
5. Файл автоматически предлагается к сохранению в браузере
6. При необходимости повторяет экспорт после изменений и повторной генерации
7. Сохранённый PDF можно открыть и распечатать
